## 2. 国标与伦理对照（**所有条款号都是 Claude 记忆，需核 PDF 原文**）

### 2.1 GB/T 39725-2020 《信息安全技术 健康医疗数据安全指南》

**⚠ 待核原文：** 以下内容是基于我对该标准的通用理解，具体条款号、分级阈值、原文措辞 **都需要你拿 PDF 核对**。

我的记忆版本里，这个标准把健康医疗数据按**三维**管：

**维度 A · 数据分类（按数据是什么）**

| 类别 | 典型数据 | 对照我们产品 |
|---|---|---|
| 1. 个人属性数据 | 姓名、身份证、联系方式、家庭地址 | `users` 表、`clientProfiles` 表 |
| 2. 健康状况数据 | 症状、诊断、测评结果、既往病史 | `assessmentResults`、`sessionNotes` |
| 3. 医疗应用数据 | 治疗方案、处方、咨询记录、AI 会话记录 | `treatmentPlans`、`episodes`、`aiCallLogs` |
| 4. 医疗支付数据 | 保险信息、账单 | 我们目前未实装 |
| 5. 卫生资源数据 | 医院资源、排班 | 我们目前未实装 |
| 6. 公共卫生数据 | 流行病学、疾病监测 | 研判分流 bucket 统计勉强算 |

**维度 B · 敏感度分级（按数据多敏感，1-5 级）**

⚠ **我对 1-5 级的具体阈值没有把握**。记忆中大致是：
- 第 1 级：可完全公开（机构公告、科普材料）
- 第 2 级：一般个人信息（已做去标识化的统计）
- 第 3 级：可识别个人的一般数据（联系方式等）
- 第 4 级：敏感个人信息（健康、生物识别、心理状况等）
- 第 5 级：极敏感（精神疾病诊断、基因、未成年人特殊信息等）

**心理咨询全文（phi_full）在国标里应该是第 4-5 级。需你拿原文核阈值。**

**维度 C · 访问主体约束**

国标通常按"医护 / 本人 / 监护 / 研究者 / 管理者 / 第三方"定义访问场景。我们的 `self_only` 对应"本人自查"、`guardian_scope` 对应"监护人代行"。

---

### 2.2 GB/T 35273-2020 《信息安全技术 个人信息安全规范》

**⚠ 待核：** 记忆中的关键概念——
- **敏感个人信息（3.2 节附近）**：14 岁以下儿童信息、生物识别、健康生理、性取向 等。**心理健康数据归入此列。**
- **去标识化（3.14 节附近）**：移除直接标识符+间接标识符，使自然人不可识别但保留重识别可能
- **匿名化（3.15 节附近）**：数据处理后无法识别且无法复原，不再属于个人信息

**对我们的意义：**
- `de_identified` 对应"去标识化"概念
- `aggregate` 的 target 应该是"匿名化"（彻底不可识别）——要严格匿名化才能摆脱个保法约束
- **`phi_full` 和 `phi_summary` 都属于"敏感个人信息"** —— 需要单独授权才能处理

---

### 2.3 中国心理学会《临床与咨询心理学工作伦理守则》(2018 修订版)

**⚠ 待核：** 记忆中的第 2 章 "保密" 有这些要点——

- **2.1 保密原则**：心理师对来访者信息负有保密义务
- **2.2 保密的限制**：来访者对自己或他人有紧迫危险时可突破（**对应我们的 L4 危机案签字流程**）
- **2.3 督导中的保密**：信息共享给督导是允许的，但督导同受保密约束（**对应 supervisor 角色看 phi_full**）
- **2.4（或其他节）专业合作中的信息共享**：跨机构转介、多学科会诊需当事人知情同意
- **2.6 记录的保管**：纸质和电子记录的安全、销毁、继承
- **2.8 针对未成年人**：原则上告知监护人必要内容，但当监护人与未成年人利益冲突时以未成年人利益优先（**对应 `guardian_scope` 的边界设计**）

**对我们的意义：**
- `guardian_scope` 不是一张纯白名单，**要能表达"特定信息对家长屏蔽"**（如未成年来访者明确要求不告知父母的内容）——当前代码没这个能力
- `supervisor` 看 phi_full 是伦理守则支持的，但**要留下访问日志**（[phi_access_logs](../../server/src/db/schema.ts) Phase 1 已加 `dataClass` + `actor_role_snapshot`）

### 2.4 《精神卫生法》 (2012 立法 / 2018 修订)

**⚠ 待核具体条款号。** 记忆中关键条款——

- **第 4 条附近**：精神障碍患者的姓名、肖像、住址、单位、身份证、健康状况及其他可推断身份的信息都受保密
- **第 23 条附近**：监护人代行权利
- **第 28-29 条附近**：诊断、治疗中监护人知情同意

**对我们的意义：**
- `phi_full` 里的危机案资料、AI 会话等一旦含精神障碍诊断线索，**保密等级应该按极敏感处理**（国标 5 级）
- 监护人知情权**不等于**对孩子所有咨询内容都有访问权——要区分"诊疗必需告知"和"完整记录"

---

### 2.5 其他可能相关（信息备查）

- **《未成年人保护法》(2020 修订)** — 第 72-74 条附近涉及未成年人个人信息保护

- **《网络安全法》** — 网络运营者对个人信息的保护义务
